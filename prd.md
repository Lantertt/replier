# **[PRD] K-Beauty 인플루언서 댓글 AI 에이전트 (가칭: K-Beauty Reply Assistant)**

## **1. 프로젝트 개요 (Overview)**

- **배경 (Why):**
    - K-뷰티 마케팅의 핵심은 틱톡/인스타그램 등 숏폼 바이럴임.
    - 인플루언서 비용이 상승함에 따라 단순 노출을 넘어 구매 전환(Sales Conversion) 을 극대화해야 함. 
    (마이크로인플루언서 팔로워 1,000명 이하의 경우 30만원, 적어도 시딩을 10명 이상하므로, 300만원 이상의 지출 발생)
    - 영상 댓글은 구매 의사가 높은 잠재 고객(Lead)의 질문(성분, 구매처, 배송 등)이 쏟아지는 곳이나, 인플루언서가 일일이 대응하기 어려워 이탈 발생.
    - 또한, 인플루언서는 인디 브랜드와 계약 시에 댓글 관리에 대해서 정확한 지침을 받지 않는 경우가 있어 댓글은 방치하는 경우가 많음
    - 인디 브랜드는 인플루언서에 비해 계약상 을의 관점에 서 있는 경우가 많아, 인플루언서에게 정확한 지침을 제공하지 않으면 광고 효율성이 줄어듬
    - 단순한 노출만으로는, ROAS (광고비 대비 매출액) 을 맞출 수가 없음
    - **댓글 대응은 단순한 CS 를 하는 것이 아니라, 구매 전환이 되기 쉬운 리드이면서 시청자 여론을 이끌어나가는 것이기에 매출 증대에 매우 중요.**

- **목표 (Goal):**
    - 인플루언서의 어투(Persona)와 브랜드의 가이드라인(Fact)을 학습한 AI가 댓글 초안을 자동 생성.
    - 인플루언서는 추천된 댓글을 확인하고, 필요 시 수정하여 댓글 대응을 최대화
    - 댓글 대응 시간을 90% 단축하고, 구매 링크 클릭률(CTR)을 높여 마케팅 ROAS(광고비 대비 매출) 개선.

## **2. 타겟 유저 (Target Persona)**

1. **인디 브랜드 마케터 (Admin):** 
제품의 소구점(USP, Unique Selling Proposition Point), 판매 링크, 금지어(과대광고 방지)를 설정하는 주체.
뷰티 인플루언서의 시딩, 사후관리를 모니터링 함으로써 ROAS 를 극대화하고자 하는 주체.
서비스를 직접적으로 구매 결정하는 주체.
2. **뷰티 인플루언서 (End-User):**
쏟아지는 댓글에 빠르게 반응하고 싶지만, 본인의 스타일(말투)을 유지하고 싶고, 시간적 제약이 있음.
실제로 제품을 리뷰하지만, 제품이 추구하는 가치, 사용성분, 제형 등에 대해서는 공부할 시간이 없음.
**인디 브랜드의 제품을 계속 시딩을 받아야 하기 때문에 평판관리가 필요함.
비록 조회수가 적게 나오더라도 댓글 관리를 열심히 할 유인이 있음.**

## **3. 핵심 기능 명세 (Key Features & Logic)**

*개발자 동료가 가장 집중해서 볼 부분입니다.*

### **기능 1: 지식 주입 (Knowledge Injection) - RAG 기반**

- **Input:**
    - **제품 정보:** 제품명, 상세페이지 텍스트(성분, 효능), 판매 링크(아마존/큐텐), 할인 코드.
    예시 제품 1 : 오브제 내추럴 커버 로션 (인스타, 틱톡, 유튜브)
    https://www.oliveyoung.co.kr/store/goods/getGoodsDetail.do?goodsNo=A000000204975
    예시 제품 2 : 메디큐브 부스터프로 (유튜브
    https://themedicube.co.kr/product/detail.html?product_no=1878&utm_source=google&utm_medium=sa_pc&utm_campaign=ager_pro&gad_source=1&gad_campaignid=20651026049&gbraid=0AAAAABTO0oT81FP8lYGUIJVsSQcak7aR_&gclid=CjwKCAiAtLvMBhB_EiwA1u6_PuohaEx_aBBdbqPrWLK5Lu94Ux1LI2GXS19eT-dRgcXUK1ejOBlxAxoCshwQAvD_BwE
    - **가이드라인:** 필수 포함 키워드(예: "진정", "물광", "), 금지 키워드(규제 리스크 방어, 예: "완치", "즉각적인 변화").
    - **페르소나(Tone & Manner):** 인플루언서의 과거 댓글 샘플 10~20개 (말투, 이모지 사용 습관 학습).
- **Process:** 브랜드가 제공한 PDF/텍스트를 벡터 DB화하여 LLM(Claude/GPT)에 참조시킴.
- **Output:** 해당 제품에 대해 정확한 정보를 말하되, 인플루언서처럼 말하는 에이전트 생성.

### **기능 2: 댓글 의도 분류 (Intent Classification)**

- **Logic:** 들어온 댓글을 LLM이 분석하여 4가지 유형으로 태깅.
    1. **구매 문의 (Lead):** "어디서 사요?", "할인 코드 있나요?" → **[우선순위 High]** 판매 링크 및 코드 포함 답변 생성.
    "지금 구매하면 이 가격이지만, 영상 하단 할인 코드는 24시간 후에 만료됩니다"와 같은 '희소성(Scarcity)' 기반의 통제 전략을 사용하여 결정을 미루는 고객들을 압박
    2. **제품 질문 (Q&A):** "민감성도 쓸 수 있나요?", "끈적이나요?" → 제품 상세 정보를 바탕으로 답변 생성.
    3. **단순 반응 (Reaction):** "언니 예뻐요", "영상 대박" → 가벼운 감사 인사 및 이모지 대응.
    4. **부정 이슈 (Risk):** "배송이 늦어요", "트러블 났어요" → **자동 답변 보류** 및 담당자 알림(Human Intervention).

### **기능 3: 댓글 생성 및 원클릭 복사 (Draft & Action)**

- **Constraint (해커톤 현실성):** 틱톡/인스타 API로 자동 게시(Auto-posting)를 구현하는 것은 권한 승인 이슈로 해커톤 기간 내 불가능할 확률이 높음.
- **Solution:** **"생성된 초안(Draft)을 보여주고 → [복사] 버튼 제공"** 또는 크롬 익스텐션 형태의 오버레이(Overlay) UI 제안.
- **UX Flow:**
    1. 인플루언서가 대시보드(혹은 앱) 접속.
    2. '답변 필요한 댓글(50개)' 리스트 확인.
    3. AI가 작성한 추천 답변 확인 (수정 가능).
    4. [복사] 후 틱톡 앱으로 이동하여 붙여넣기 (또는 링크 연동).

## **4. 데이터 흐름도 (Data Flow Architecture)**

*동료에게 이 구조를 그려주거나 설명해주세요.*

1. **Data Collection:** (크롤링 또는 API) 특정 영상의 댓글 수집.
2. **Preprocessing:** 광고성 스팸 댓글 필터링.
3. **LLM Processing:**
    - Prompt: "너는 20대 K-뷰티 인플루언서야. 아래 브랜드 가이드(Context)를 참고해서, 사용자의 질문(Query)에 대해 친근한 말투(Persona)로 답해줘. 단, 의학적 효능은 과장하지 마(Safety)."
4. **Client Side:** 웹/앱 화면에 원문 댓글과 AI 답변 초안을 1:1 매칭하여 출력.

## **5. 차별화 포인트 (해커톤 심사 포인트)**

- **규제 준수 (Compliance):** K-뷰티 수출 시 가장 중요한 것이 **과대광고 규제(FDA, MoCRA)**임. AI가 답변할 때 "여드름 치료(Cure)" 같은 금지어를 "진정(Soothing)"으로 순화해서 답변한다는 점을 강조 (Safety Layer).
- **세일즈 연동:** 단순 소통이 아니라, **"구매처 링크 + 할인 코드"**를 문맥에 맞게 자연스럽게 삽입하여 **마케팅 효율(Conversion)**을 높인다는 비즈니스 임팩트 강조.

## **6. 리스크 관리**

### **1. 할루시네이션 및 성분 오류 리스크 (Trustworthiness)**

화장품은 피부에 직접 닿는 제품이므로 정보의 정확성이 무엇보다 중요하다. AI가 성분을 잘못 설명하거나 맞지 않는 피부 타입을 추천할 경우 법적 분쟁으로 이어질 수 있다.

- **솔루션**: '근거 기반 답변(Grounded Response)' 시스템을 구축한다. AI가 답변을 생성할 때 참조한 제품 상세페이지의 특정 단락을 인플루언서에게 같이 보여주어 검수 속도를 높인다. 또한, 정보가 불확실할 경우 억지로 답변하지 않고 "이 부분은 제가 담당자님께 한 번 더 확인해보고 말씀드릴게요!"와 같은 '인간 개입형 유도 멘트'를 출력한다.

### **2. 인플루언서-팬 관계 훼손 우려 (Authenticity)**

팬들은 인플루언서와의 '직접적인 소통'을 기대하며 댓글을 단다. AI가 답변한다는 사실이 알려질 경우 진정성이 훼손될 수 있다는 피드백이 존재한다.

- **솔루션**: '인플루언서 최종 승인 시스템(Human-in-the-Loop)'을 기본값으로 설정한다. AI는 '작성자'가 아닌 '비서(Sidekick)'로서 초안을 제공하고, 인플루언서가 자신의 손으로 '게시' 버튼을 누르게 함으로써 소통의 주체성을 유지한다. 또한, AI 답변의 퀄리티를 높이기 위해 인플루언서가 평소 라이브 방송에서 했던 발언이나 유튜브 영상 자막 데이터를 추가로 학습시켜 '진짜 인플루언서가 할 법한 말'을 생성한다.

## **7. 개발 우선순위 (MVP Scope)**

1. **(Must Have)** 브랜드 정보(텍스트) 입력 시 RAG를 통해 답변 생성하는 백엔드 로직.
2. **(Must Have)** 댓글 유형(구매/질문/비난) 분류 로직
3. (Must Have) 댓글 유형에 대해서 브랜드 정보 + 인플루언서 스타일에 따라서 추천 답변 생성
: 유튜브는 댓글 API 가 열려 있으니, 무조건 구현하기
: 인스타와 틱톡은 내 계정으로 구현해보기 (내 계정에는 답변이 없으니까, 브랜드 정보로 가는 것) 
4. **(Nice to Have)** 실제 틱톡/인스타 UI처럼 보이는 데모 화면 (API 연동이 안 되더라도 UI상으로 그럴듯하게 보여주는 것이 중요).

### **💡 기획자 팁 (동료에게 건넬 말)**

"OO님, 기술적으로 완벽한 자동화보다는 **'브랜드가 준 정확한 정보(Fact)'**와 **'인플루언서의 말투(Tone)'**가 섞이는 그 **'결합(Align)'**을 보여주는 게 핵심이야. API 연동이 막히면 무리하지 말고, 우리가 가상의 댓글 데이터를 넣었을 때 AI가 기가 막히게 답변을 뽑아내는 **'데모 화면'** 위주로 구현하자."

이상적인 제품은, 댓글 추천이 나오면 댓글을 조금 수정하고 바로 게시를 누르면 자동으로 게시가 되는 것 (1 게시물 당 1 권한)

그러나 이걸 MVP 에는 못 만들 것이기 때문에, 댓글 추천만 해주는 걸로 해서, 그걸 ctrl c/v 해서 게시하는 방법이 맞겠음. 이러면 대충 댓글 50개 정도는 처리할 수 있을 것임. 그러면 팔로워 1,000명 미만 마이크로 인플루언서에는 대응 가능.
